# Security Policy

## Supported Status

This repository is in early development. Until versioned releases exist, security fixes will be handled on a best-effort basis.

## Reporting A Vulnerability

Please do not disclose security issues in a public GitHub issue.

Preferred approach:

1. Use GitHub's private security reporting or security advisory flow if it is enabled for this repository.
2. If that is not available, contact the repository owner directly through GitHub.
3. Include enough detail to reproduce and assess the issue safely.

Helpful information:

- affected file or area
- reproduction steps
- impact assessment
- whether credentials, tokens, or order placement could be involved
- suggested mitigation, if known

## Sensitive Areas

Please take extra care with reports involving:

- OAuth credentials or token handling
- session TAN or transaction authentication flows
- order placement or cancel/change flows
- logging of sensitive account or trading data
- sandbox versus production environment separation

## Disclosure Expectations

Please allow reasonable time for investigation and remediation before any public disclosure.
